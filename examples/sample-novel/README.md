# 示例项目 · 《山河入梦》

这是 InkForge 自带的最小化示例：一个项目元信息 (`book.json`) + 一章示例正文 (`chapters/01-楔子.md`)，大约 200 字，既可直接浏览，也可以"导入"后接着写。

## 怎么用

### 方案 A：直接复制到你的工作目录

1. 启动 InkForge，完成首次运行向导（创建一个空项目）
2. 关闭应用
3. 把本目录复制到你的 `<workspace>/projects/` 下，重命名为你喜欢的项目名
4. 重开 InkForge，新项目会出现在侧栏（M5-B 欢迎向导完善后会支持 UI 内直接"从示例导入"）

### 方案 B：手动新建 + 粘贴章节

1. InkForge 新建一个空项目
2. 新建章节「楔子 · 起风」
3. 把 `chapters/01-楔子.md` 的正文粘贴进去保存
4. 完成：此时你可以：
   - 选中任意文字 → 选段工具条的"润色/审查/续写/代入"
   - 切到 🎭 酒馆（Ctrl+4）创建林晚 / 陆九两个角色卡，再围绕「是否接下任务」让 AI 互演
   - 切到 🌍 世界观（Ctrl+5）创建条目"青松门"、"师门簿册"
   - 切到 📊 审查（Ctrl+7）跑一次 5 维审查

## 为什么只有 200 字？

示例的重点不是故事，而是**让新用户立即产生第一次 AI 反馈**。超过 200 字时 `analysis-service` 的默认 `every-n-chars` 会跑，时间线会冒出来第一条，验证整条链路通畅。

继续写作或删掉示例章节都不会影响系统。

## 文件说明

| 文件 | 作用 |
|------|------|
| `book.json` | 项目元信息，冗余镜像 SQLite 中的 projects/chapters 行；便于 Git 或 Obsidian 协同 |
| `chapters/01-楔子.md` | 章节正文（Markdown）|
